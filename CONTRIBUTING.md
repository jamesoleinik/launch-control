# Contributing to Launch Control

Thanks for your interest! This project is designed to be forked and adapted for any cross-team business process.

## How to Adapt for Your Scenario

1. **Fork this repo**
2. **Replace the data model** — swap Launches/Milestones/Tasks for your domain (e.g., Onboarding Stages, Release Phases, Incident Categories)
3. **Rewrite the business skills** — encode YOUR team's process in `business-skills/`
4. **Update the scripts** — point the Python/CLI scripts at your tables
5. **Reskin the dashboard** — customize the generative Power Apps page (Ep 9) for the views your team cares about

The three-layer pattern stays the same. Only the business logic changes.

## Reporting Issues

Found a bug or have a suggestion? [Open an issue](https://github.com/jamesoleinik/launch-control/issues) — there are templates for bugs, questions, and episode suggestions.

## Code of Conduct

This project follows the [Microsoft Open Source Code of Conduct](CODE_OF_CONDUCT.md). Be kind, be constructive — we're all building together.
